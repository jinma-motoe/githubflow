AMD Git Flow

この手順書は AMDで Git を使用する場合、参考するために作られた手順書です。このページも Github でソースコードが確認できます。何か修正したいところがあればいつでもお願いします。

  1. PULL
  2. BRANCH
  3. EDIT
  4. COMMIT
  5. PULL REQUEST
  6. MERGE
  7. RELEASE

BRANCH

安全な作業のため、マスターから BRANCH を作り、その BRANCH にコードを書きます。
BRANCH 名は作業目的を表すアルベルト表記で入力します。
例：about-page-tsukuru, about-page-naosu

EDIT

自分のブランチで安全にソースコードを作ったり、直したりします。

COMMIT

どんな作業をしましたか。他の人がわかるように変更したコードの記録を残します。

PULL REQUEST

PULL REQUEST を使って自分のコードレビューをマスターに申請し、問題がなければブランチのコードがマスターに反映されます。
PULL REQUEST する場合、18時までに PULL REQUEST してください。

MERGE

こうやって PULL REQUEST されたブランチはマスター側でブランチをに合体させます。

RELEASE

ソースコードを FTP サーバーに公開する段階です。
